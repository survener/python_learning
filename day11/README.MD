# Day11 - Goal

## Main Goal
Build a clean Python package structure and understand why modular layout improves maintainability.

## Learning Objectives
1. Understand package layout (`myapp/`, `__init__.py`, module files).
2. Split logic into small modules instead of one large script.
3. Use clear imports between modules.
4. Keep code testable by separating business logic from CLI/input handling.

## Practical Targets
1. Create a small package `day11/myapp`.
2. Implement one core module (logic) and one interface module (runner).
3. Add at least 3 tests that validate package behavior.
4. Make sure `pytest -q day11` passes.

## Git Targets
1. Work on `feature/day11-structure`.
2. Commit in small, meaningful steps.
3. Merge back to `main` only after tests pass.

## Done Criteria
- Package structure is complete and readable.
- Tests pass.
- Branch history is clean.
- Changes are pushed to GitHub.
