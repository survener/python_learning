# Day16 - Goal

## Main Goal
Improve test depth by adding edge-case coverage to existing modules.

## Learning Objectives
1. Identify weak spots in existing tests.
2. Add tests for invalid input, boundaries, and special cases.
3. Improve confidence without changing application behavior.
4. Keep refactor-free test additions clean and targeted.

## Practical Targets
1. Add more tests to `day08/test_todo_cli.py`
2. Add more tests to `day09/test_todo_store.py`
3. Add more tests to `day11/test_core.py`
4. Run all related tests successfully

## Done Criteria
- New edge-case tests are added
- All tests pass
- Changes are committed on a feature branch
